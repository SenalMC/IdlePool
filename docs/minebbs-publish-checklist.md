# MineBBS 发布前检查清单

## 需要作者决定

- [ ] 确认最终资源名称是否继续使用 `IdlePool`。
- [x] 填写作者 Chirnuo、联系邮箱和 GitHub 反馈渠道。
- [ ] 选择免费、金粒或 RMB 发布方式。
- [x] 使用 Apache License 2.0，并通过 NOTICE 要求再分发与修改版保留来源署名；修改版不强制开源。
- [ ] 决定是否公开说明开发过程中使用了 AI 辅助。
- [ ] 决定是否把 `rc.3` 作为公开测试版，还是完成更多版本实测后改为 `1.0.0`。

## 页面字段建议

- [ ] 分类选择 `Java版服务器资源 → 服务器插件 → 服务端插件`。
- [ ] 版权类型选择 `原创`。
- [ ] 版本填写 `1.0.0`。
- [ ] 标题明确写出 Paper、挂机池、时间里程碑、暂存箱等关键功能。
- [ ] 必需前置填写“无”，可选前置单独列出。
- [ ] 不要将 Paper 26.2 写成“已实测”；目前完整实测版本为 Paper 1.21.1 build 133。

## 展示素材

- [ ] 制作一个清晰的方形资源图标，建议至少 512×512。
- [ ] 截取玩家开始挂机的原生箱子 GUI。
- [ ] 截取挂机中的 ActionBar。
- [ ] 截取奖励时间参数管理 GUI。
- [ ] 截取暂存箱领取界面。
- [ ] 截取 `/afkpool doctor` 的正常检查结果。
- [ ] 截图中隐藏服务器 IP、玩家隐私、后台路径和敏感插件信息。

MineBBS 的相关资源规范要求资源进入资源中心、选择正确分类，并建议清楚说明前置与兼容信息。发布前再次阅读：[Java 服务端插件区资源发布规范](https://www.minebbs.com/threads/java.23025/) 与 [内容版权规范](https://www.minebbs.com/threads/minebbs.353/)。

## 上传文件

- [ ] 主附件上传 `IdlePool-1.0.0-release.zip`。
- [ ] 可选单独上传 `IdlePool-1.0.0.jar`，方便不使用 ItemsAdder 的服主。
- [ ] 确认压缩包大小没有超过 MineBBS 当前附件限制。
- [ ] 下载本地附件后重新解压，确认 JAR、IA 内容和文档完整。
- [ ] 在资源页面填写 SHA-256，方便用户校验文件。
