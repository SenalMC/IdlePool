# 开始与结束挂机

## 开始挂机

玩家进入已启用的挂机池后，插件会自动打开原生箱子 GUI。进入区域本身不会获得奖励，必须点击“开始挂机”。

开始前会检查：

- 挂机池是否启用
- 玩家是否位于区域内
- 玩家是否拥有挂机池权限
- 当前挂机人数是否达到上限
- 是否已有活动会话

开始成功后，插件从 SQLite 读取未满周期进度，并丢弃已经超过保留期的旧进度。

## 状态显示

默认 ActionBar 显示：

- 本次连续挂机时间
- 距离下次周期奖励的进度
- 本次已获得奖励数量

管理员也可以在 `config.yml` 中启用 BossBar。倍率活动生效时，`{event}` 与 `{multiplier}` 可显示活动名称和进度倍率。

## 结束条件

以下情况会结束本次挂机：

- `/afkpool stop`
- 离开挂机池区域
- 下线
- 死亡
- 服务端关闭或插件停用

本次连续挂机里程碑会重置，但未满普通周期的进度按 `progress-retention` 保留。

正常主动结束或离开区域后，插件默认自动打开一次暂存箱。可通过 `inbox.open-on-stop` 关闭。
